# TaskSphere Product Backlog

Here is the sequence of user stories and DevOps tasks for the project:

1. **Environment Variables** (Priority: High)
   As a dev, I want to configure environment variables so we don't commit secrets like API keys to git.

2. **Registration** (Priority: High)
   As a user, I want to register for an account so that I can start using TaskSphere.

3. **Login** (Priority: High)
   As a user, I want to log in to my account so I can see my private tasks.

4. **Logout** (Priority: High)
   As a user, I want to log out so that other people can't use my account.

5. **Create Task** (Priority: High)
   As a user, I want to create new tasks so that I don't forget them.

6. **View Tasks** (Priority: High)
   As a user, I want to view my tasks in a list so I know what I have to do.

7. **Update Task** (Priority: Medium)
   As a user, I want to edit my tasks so I can update their status or description.

8. **Delete Task** (Priority: Medium)
   As a user, I want to delete tasks so I can clean up my list.

9. **Dockerize backend** (Priority: Medium)
   As a dev, I want to run the backend in a Docker container so that it works the same way for everyone.

10. **Dockerize frontend** (Priority: Medium)
    As a dev, I want to dockerize the frontend so it is easy to build and run.

11. **GitHub Actions** (Priority: Medium)
    As a dev, I want to set up GitHub Actions so our tests run automatically when we push code.

12. **Pull Request Workflow** (Priority: Medium)
    As a dev, I want to configure a pull request workflow so we have to check tests and get reviews before merging to main.

13. **CircleCI** (Priority: Low)
    As a dev, I want to configure CircleCI to run checks as an alternative CI tool.
